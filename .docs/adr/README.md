# ADRs (Architecture Decision Records)

Use this folder for durable technical decisions.

## Naming
`YYYY-MM-DD-short-title.md`

## Template
1. Context
2. Decision
3. Alternatives considered
4. Consequences
5. Status (Proposed/Accepted/Deprecated)

## When to add an ADR
1. Stack or framework changes.
2. Auth or security policy changes.
3. Data model changes with migration impact.
4. Any decision likely to be revisited later.
